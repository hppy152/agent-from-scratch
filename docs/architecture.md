## Agent 的灵魂：感知-思考-行动循环

```
        ┌──────────────────────────────────────────┐
        │              Agent 核心                    │
        │                                           │
        │   ┌─────────┐                             │
  输入 →│   │  感知层  │  接收用户输入、环境信息       │
        │   └────┬────┘                             │
        │        ↓                                   │
        │   ┌─────────┐                             │
        │   │  思考层  │  LLM 推理、规划、决策        │
        │   └────┬────┘                             │
        │        ↓                                   │
        │   ┌─────────┐                             │
        │   │  行动层  │  调用工具、执行操作           │
        │   └────┬────┘                             │
        │        ↓                                   │
        │   ┌─────────┐                             │
        │   │  记忆层  │  短期记忆 + 长期记忆         │
        │   └─────────┘                             │
        │                                           │
        └──────────────────────────────────────────┘
```

## 各层详解

### 1. 感知层（Perception）

```
接收输入 → 分类（文本？指令？工具结果？）→ 格式化为消息
```

这是 Agent 与外界的接口。它接收：
- 用户的文本输入
- 工具执行的结果
- 其他 Agent 的消息

### 2. 思考层（Reasoning）

```
LLM + System Prompt + 对话历史 + 工具定义 → 决策
```

这是 Agent 的大脑。它做三件事：
- **理解**：用户到底想要什么？
- **规划**：需要几步？该用什么工具？
- **推理**：根据已有信息，下一步该怎么做？

### 3. 行动层（Action）

```
LLM 的决策 → 选择工具 → 传递参数 → 执行 → 返回结果
```

这是 Agent 的手脚。它执行 LLM 决定的操作：
- 调用外部 API
- 读写文件
- 计算、搜索、生成内容

### 4. 记忆层（Memory）

```
短期：当前对话的消息列表
中期：对话太长时压缩为摘要
长期：重要事实保存到文件
```

这是 Agent 的记忆系统。让 Agent 能：
- 记住当前对话的上下文
- 跨对话保持连续性
- 积累经验和知识

## 信息流：一次完整的交互

```
用户: "帮我算一下 3 的 17 次方"

                    ┌──────────────┐
                    │   感知层     │
                    │  接收输入     │
                    └──────┬───────┘
                           ↓
                    ┌──────────────┐
                    │   思考层     │
                    │  LLM 推理:   │
                    │  "这是数学题， │
                    │   需要计算"   │
                    └──────┬───────┘
                           ↓
                    ┌──────────────┐
                    │   行动层     │
                    │  调用        │
                    │  calculate(  │
                    │  "3**17")    │
                    └──────┬───────┘
                           ↓
                    ┌──────────────┐
                    │   记忆层     │
                    │  记住:       │
                    │  用户问了     │
                    │  3**17       │
                    └──────┬───────┘
                           ↓
                    "结果是 129140163"
                           ↓
                        输出给用户
```

## ReAct 循环：多步推理

```
        ┌──────────────┐
        │  用户输入     │
        └──────┬───────┘
               ↓
     ┌─────────────────┐
     │   思考 (Think)   │ ←──┐
     │   分析当前状态    │    │
     └──────┬──────────┘    │
            ↓               │
     ┌─────────────────┐    │
     │   行动 (Act)     │    │
     │   调用工具        │    │ 循环
     └──────┬──────────┘    │
            ↓               │
     ┌─────────────────┐    │
     │   观察 (Observe)  │ ───┘
     │   获取结果        │
     └──────┬──────────┘
            ↓ (任务完成)
     ┌─────────────────┐
     │   最终回答        │
     └─────────────────┘
```

## 多 Agent 协作

```
              ┌───────────────┐
              │   Orchestrator │
              │   (指挥官)     │
              │               │
              │  1. 理解需求   │
              │  2. 分解任务   │
              │  3. 分配给worker│
              │  4. 汇总结果   │
              └───────┬───────┘
                      │
         ┌────────────┼────────────┐
         ↓            ↓            ↓
   ┌───────────┐ ┌───────────┐ ┌───────────┐
   │ Worker A  │ │ Worker B  │ │ Worker C  │
   │ (搜索)    │ │ (计算)    │ │ (写作)    │
   └───────────┘ └───────────┘ └───────────┘
```
