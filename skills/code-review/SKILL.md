# 代码审查技能

## 什么时候用
当用户请求代码审查、优化建议、或代码质量分析时使用。

## 审查维度
1. **正确性**：代码逻辑是否正确？有没有 bug？
2. **可读性**：命名是否清晰？注释是否充分？
3. **性能**：有没有明显的性能问题？
4. **安全性**：有没有安全漏洞？
5. **最佳实践**：是否遵循语言/框架的最佳实践？

## 输出格式
```
## 代码审查报告

### 优点
- ...

### 问题
- 🔴 严重: ...
- 🟡 建议: ...

### 优化建议
1. ...
2. ...
```

## 注意事项
- 先肯定优点，再指出问题
- 给出具体的修改建议，不要只说"这里不好"
- 区分"必须改"和"建议改"
